# github-notify 手動 QA チェックリスト

## 使い方

- リリース前に本チェックリストを上から順に実施する。
- `結果(Pass/Fail)` は実施者が記入する。
- `備考` には再現条件、ログ位置、スクリーンショット保存先を記録する。
- `自動E2E対応` は該当する `e2e/scenarios/fxx-*.spec.ts` を示す。

---

## 前提条件

| 項目 | 内容 |
|---|---|
| ビルド | 開発版またはリリース候補版をインストール済み |
| GitHub認証 | テスト用アカウントを用意 |
| 通知データ | 未読通知、レビュー依頼、メンションを含む状態が望ましい |
| OS権限 | 通知許可/拒否の両パターンを試せること |
| ネットワーク | オンライン/オフラインを切り替え可能 |
