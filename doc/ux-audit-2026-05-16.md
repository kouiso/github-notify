# UX/UI Audit — github-notify (2026-05-16)

**監査者**: Uchida Yuki (Claude Opus 4.7)
**対象 commit**: `origin/main` @ `b9bc0e6`
**手法**: 静的コードレビュー(Tauri デスクトップアプリで WSL ヘッドレス環境のため Playwright/実機巡回は非適用。コード読み + アクセシビリティ静的解析で代替)
**検証ソース**: `[コード解析]`

## サマリー

`github-notify` は React 19 + Tauri 2 + Tailwind CSS 4 で組まれた GitHub 通知デスクトップアプリ。設計の骨格は良く、shadcn 系の component 構造と CSS variable ベースの design token が整理されている。ただし **a11y(キーボード操作・スクリーンリーダ・コントラスト) の積み残しが多く**、特に「マウス hover に依存した UI」「modal の focus 管理欠落」「`<div role="button">` の濫用」「`prefers-reduced-motion` 未対応」が WCAG AA 観点で課題。i18n の混在(ja/en)、formatRelativeTime の重複定義、Dialog の Escape ハンドラのバグなど、設計負債も複数。

Round 1 で **34 件** の issue を検出 (Critical 7 / High 11 / Medium 11 / Low 5)。1 件 (M3) は後の bot review で false positive と判明し撤回 → 実効 33 件。
Round 2 で +17 件 (Critical 1 / Warning 9 / Tip 7、P2 を後で Tip に downgrade)、Round 3 で +11 件 (Warning 6 / Tip 5) を追加し、**累積 61 件** (Blocker 6 / Critical 2 / Warning 35 / Tip 18、claude-design-auditor rubric)。
